# Обозреватель BTC RPC

[![версия npm][npm-ver-img]][npm-ver-url] [![загрузки NPM][npm-dl-img]][npm-dl-url]


Простой обозреватель биткойн-блокчейнов без базы данных через RPC. Создан с помощью Node.js, экспресс, bootstrap-v4.

Этот инструмент задуман как простой самостоятельный обозреватель блокчейна Биткойн, управляемый вызовами RPC к вашему собственному узлу bitcoind. Этот инструмент прост в использовании, но в настоящее время ему не хватает функций по сравнению с проводниками на основе базы данных.

Я создал этот инструмент, потому что хотел использовать его сам. Какими бы ни были причины запуска полного узла (ненадежность, технический интерес, поддержка сети и т. д.), полезно оценить «полноту» вашего узла. С помощью этого обозревателя вы можете не только исследовать блокчейн (в традиционном понимании термина «проводник»), но и исследовать функциональные возможности собственной ноды.

Живые демонстрации доступны по адресу:

* Биткойн: https://btc.chaintools.io
* LTC: https://ltc.chaintools.io

# Функции

* Просмотр блоков
* Просмотр сведений о блоке
* Просмотр сведений о транзакциях с навигацией «назад» по проведенным транзакциям
* Просмотр содержимого JSON, используемого для создания большинства страниц
* Поиск по идентификатору транзакции, хешу/высоте блока и адресу
* Необязательная история транзакций для адресов путем запроса настраиваемых серверов ElectrumX.
* Сводка мемпула с разбивкой по комиссиям, размеру и возрасту
* Браузер команд RPC и терминал
* В настоящее время поддерживает BTC, LTC (можно легко добавить поддержку любой монеты, совместимой с протоколом Bitcoin-RPC)

# Начиная

Приведенные ниже инструкции ориентированы на BTC, но их можно легко адаптировать и к другим монетам.

## Предпосылки

1. Установите и запустите полноценную архивирующую ноду - [инструкции](https://bitcoin.org/en/full-node). Убедитесь, что на вашем биткойн-узле включена полная индексация транзакций (`txindex=1`) и включен сервер RPC (`server=1`).
2. Синхронизируйте свой узел с сетью Биткойн.
3. «Последняя» версия Node.js (рекомендуется 8+).

## Инструкции

``` ударить
npm установить -g btc-rpc-explorer
btc-rpc-explorer
```

Если вы работаете в основной сети с каталогом данных и портом по умолчанию, это должно просто работать.
Откройте [http://127.0.0.1:3002/](http://127.0.0.1:3002/) для просмотра проводника.

Вы можете установить параметры конфигурации в файле `.env` или с помощью аргументов CLI.
Подробнее см. [configuration](#configuration).

### Конфигурация

Параметры конфигурации могут быть переданы как переменные среды
или создав файл env в `~/.config/btc-rpc-explorer.env`
или в `.env` в рабочем каталоге.
См. [.env-sample](.env-sample) для списка опций и деталей форматирования `.env`.

Вы также можете передавать параметры в качестве аргументов CLI, например:

``` ударить
btc-rpc-explorer --port 8080 --bitcoind-port 18443 --bitcoind-cookie ~/.bitcoin/regtest/.cookie
```

См. `btc-rpc-explorer --help` для получения полного списка опций CLI.

## Запустить через Docker

1. `docker build -t btc-rpc-explorer.`
2. `docker run -p 3002:3002 -it btc-rpc-explorer`
